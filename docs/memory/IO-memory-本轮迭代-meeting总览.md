## 这轮做了什么(已完成)

1. **记忆能被 agent 自己查(agentic 召回)**
   agent 先看「目录」、挑相关的、再取「正文」回答。比以前的纯关键词更准。
2. **任何模型都不丢召回(兜底)**
   route B 用的是用户自己的模型,有的弱模型不会按格式调工具 → 服务端自动用算法补召回。**这是关键:不赌模型自觉。**
3. **修了一个召回 bug**
   模型会被自己刚说错的话带偏(明明记得却说"没有")→ 已修(纠正时让记忆优先于刚才的错话)。
4. **记忆能写、能纠正(M2)**
   - 新事实 → 写成一张卡;
   - 用户纠正旧事实 → 写新卡 + 旧卡**当场退场**(标记"已被取代",指向新卡),**永不硬删、可追溯**;
   - 记忆卡第一次有了「重要度 / 状态 / 敏感标记」这些属性。
5. **iOS 花园不受影响**
   新卡同时回写老字段,Garden 显示照旧、不用改前端。

## 下一步重点:M3 的起点是一个真 bug

**现象**:用户聊天里说「我养了条狗叫蛋子、是比熊、屁股有胎记」——记录里有,但**没被记住**。
**根因**:不是存储坏了,是后台「判断该不该记」的环节**偏向记"偏好(我喜欢…)",漏掉"事实(我养了狗…)"**。
**M3 要做**:用 **eval(可重复测试)驱动**,把"该记什么"调对——
1. 先立一组测试题(该记 / 不该记 / 该纠正 / 该承认不知道);
2. 把记忆判断从"只记偏好"**扩到记事实**(宠物/人/地点/物品/关键生活事实);
3. 防止"记太多 → 把幻觉/临时内容也记进去"。

> **命门**:记多(抓到狗)和记少(防污染)是一对矛盾,只能靠 eval 量着调。
