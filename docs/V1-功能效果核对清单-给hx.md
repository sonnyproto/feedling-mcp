> 每条:**【功能】这功能该干什么 → 【怎么触发】→【✅预期效果/成功标准】→【❌坏了长这样】**。

## 1. 新用户 onboarding(genesis 蒸馏)

**这功能该干什么**:新用户把历史/角色卡/资料喂进来 → 在 CVM 里蒸馏出**三层:声音(voice)+ 身份(identity)+ 记忆(memory)**。
**怎么触发**:开 Genesis 开关(DebugTool 里) → 全新账号走 onboarding → 导入历史 → 提交。
**✅成功**:进度走完进聊天,且**三层都到位** —— 回复像本人(voice)、身份卡有名字/维度(identity)、问历史里的人/地点答得上(memory)。
**❌坏**:卡死/报错;或进去后没性格、身份空白、问历史一问三不知(= 蒸馏没产出/没接上)。
📊 **面板**:出现 `genesis.outputs.applied`(detail 里 source_kind + memory_action_count/identity_status/persona_ref)= 蒸馏真落地了。

## 2. Voice / persona 注入

**这功能该干什么**:每次对话 spawn 时,把蒸出来的 persona 注入,让 agent 用**这个用户专属的声音**说话(不是通用助手)。
**怎么触发**:onboarding 完之后正常聊。
**✅成功**:语气/自我认知稳定符合 persona,多轮不串味、不退回通用腔。
**❌坏**:回复像随便一个通用助手,完全没那个"人设"。

## 3. Identity 身份卡

**这功能该干什么**:存这个 AI 的身份(名字/维度/自我介绍),开机注入,长期稳定;老用户的身份不被覆盖。
**怎么触发**:进身份页看;老账号登录看。
**✅成功**:新用户身份由 genesis 蒸出且有内容;**老用户原有身份原样保留、不被洗掉**。
**❌坏**:身份空白/崩;或老用户身份被重置成空/默认。

## 4. Memory 写 / 读 / 召回

**这功能该干什么**:对话里的持久事实自动落成 v1 记忆卡;之后能按需召回用上;能按 bucket/线索精确查。
**怎么触发**:聊天说一个事实("我家狗叫蛋子")→ 过会儿问它("我家狗叫啥?")。
**✅成功**:事实被记住(落卡)、之后问得出(召回);Garden 不崩、能看到这张 v1 卡的核心内容。
**❌坏**:没落卡 / 问不出 / Garden 崩或显示坏块。
📊 **面板**:说事实那轮出现 `memory.write.actions`(types 含 memory.add、status=ok)= 落卡了;问它那轮出现 `memory.index.called` / `memory.fetch.called`(detail 有 counts)= 真去召回了。
> 注:Garden 的分类/线索好不好看 = 体验验收,**不挡后端 v1 上线**(当前 Garden 可能还是简版/给设计看的页)。红线只是"不崩 + 看得到卡内容"。

## 5. host → agent_runtime 切换(本版核心,非干净切)

**这功能该干什么**:把 host 聊天从老的 route-B 切到 agent_runtime;但**图片和 gateway provider 故意保留老路**,不能切断;identity 写读独立、不受影响。
**怎么触发**:正常文本聊天 / 发图片 / 用 gateway provider 的账号聊 / 期间改身份。
**✅成功**:文本走新 runtime 正常;**发图片照样能聊(没断)**;**gateway 账号没断**;身份写读一切正常。
**❌坏**:发图片报错/断;gateway 账号断;身份功能受影响。
📊 **面板**:纯文本那轮出现 `route.decided mode=agent_runtime`;发图片那轮出现 `route.decided mode=legacy reason=image`;gateway 关时 `mode=legacy reason=driver_legacy` —— 一眼看出走了新路还是回落 legacy。
> 红线:图片断、gateway 断 = 直接别上。

## 6. 老用户 voice backfill(不退化)

**这功能该干什么**:上线前的老用户没有蒸馏出的 persona,切过来后**自动补**,不让他们变通用助手。
**怎么触发**:老账号登录 → 正常聊几句。
**✅成功**:头一两条可能偏通用,**之后恢复**原性格;记忆也还在。
**❌坏**:老账号永久变通用、性格丢、记忆没了。
> 注:要后端开了 backfill;没立刻生效先告诉我(可能 ops 没开),不一定是 bug。

## 7. Proactive 主动陪伴

**这功能该干什么**:在合适时机(冷场/特定窗口)主动找用户,内容来自记忆/感知,不骚扰。
**怎么触发**:放着别动一段时间。
**✅成功**:适时收到合理的主动消息,不刷屏、不半夜乱发。
**❌坏**:狂发 / 半夜乱发 / **确认开关已开、且到达测试窗口后仍不发**。
> 注:proactive 依赖后端 Dream/wake 定时器 + ops flag + 时间窗口。**没动静先确认 ops 开没开、是否到窗口,别把"没开"当功能坏**(这条不能在 ops 未开时算 ❌)。

## 8. 感知(若 v1 开了这块)

**这功能该干什么**:基于场景(家/公司/锁屏等)做合理感知,喂给陪伴,且不泄隐私。
**✅成功**:场景相关的表现合理、隐私不外泄。
**❌坏**:感知乱套 / 隐私泄漏。

## 9. 数据安全(删号 / 多租户)

**这功能该干什么**:删数据/重置要把旧身份/记忆/上传任务清干净,新号不串到旧号。
**怎么触发**:删除我的数据 → 再注册新号。
**✅成功**:新号干净开始,绝不带上一个账号的任何东西。
**❌坏**:新号还带旧账号的身份/记忆 = 严重,必报。

- 5:图片断 / gateway 断
