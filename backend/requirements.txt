# Production WSGI server for both the Flask backend and the enclave service.
# The dev server in app.run() (Werkzeug) is single-process and stalls on large
# responses under the TDX CVM; gunicorn gives worker timeouts + better
# write-out. The backend MUST stay single-worker: app.py starts the WebSocket
# ingest thread at import time (binds :9998), so multiple workers would collide
# on it. The enclave embeds gunicorn programmatically (BaseApplication) from
# enclave_app.py's __main__ so its compose entrypoint stays
# `python -u backend/enclave_app.py` (compose_hash unchanged); it serves its
# in-enclave TLS via the gunicorn `ssl_context` hook (needs >=21.0).
